# CCHub 成色评级标准 v1.0

> 人有段位,货有成色。CCHub 注册表里的每一件货,标注的不是"我们觉得它多好",而是**我们为它担保到什么程度**——档位越高,我们替你验得越深。
> 本标准公开(CC BY 4.0),升降档在仓内留痕可查。

## 六档(对用户可见)

| 档 | 名 | 你可以怎么理解 | 进档硬标准 | 谁有权定 |
|---|---|---|---|---|
| **SSS** | 镇馆 | 我们敢拿声誉押注 | 团队日常工作流**离不开它**≥3个月 + before/after 效果实证 + 有社区真实使用反馈 + 全队署名推荐 | Founder 亲批,全馆 ≤3 |
| **SS** | 常驻 | 我们自己天天在用 | 真装真用后**决定留用**≥2周 + 写出 before/after 对比 + 人工过码 + license 亲验 | Curator 提名 + dogfood 记录 |
| **S** | 放心装 | 我们替你完整验过一遍 | 三道闸全过 + **真装真跑一次完整流程** + license 亲验 + 人工逐行过码无恶意 | Curator |
| **A** | 精选 | 机器和人都粗验过,没红旗 | 四道硬筛过 + clone 红旗装测干净 + **license 亲验**(不是声称) + 自包含确认 | Tester |
| **B** | 优选在册 | 高热度+机器扫描干净 | ⭐≥300(或自媒体双信号) + 机器红旗扫描无命中 + license 声称兼容 | 机器+Tester 复核 |
| **C** | 在册 | 值得关注,尚未验证 | 类型是五轴单品 + 定位到源仓 + license 声称非 NC/ND | 机器 |

**看不到的那一档**:未过合规/安全/品味任一闸的,内部留档原因,**不公开点名**——我们只策展好的,不羞辱差的。

## 规则

1. **升档靠证据**:C→B 要红旗扫描,B→A 要人工装测,A→S 要真用,S→SS 要留用,SS→SSS 要时间。每次升档在 git 留痕:谁、何时、依据什么。
2. **降档自动触发**:停止维护>6个月 / license 变更收紧 / 爆出安全问题 → 立即降档或下架并公告。
3. **机器的权限到 B 为止**:S 及以上必须过人手,这是「放心装」三个字的可信度来源。
4. **数量纪律**:SSS ≤3、SS ≤10。宁可空着,不凑数。
5. **利益回避**:CCHub 自己产的东西不参与评级(单独标「CCHub 出品」)。
